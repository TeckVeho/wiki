# 🚀 Cursor 利用ガイドライン

## 1. はじめに

このガイドラインは、AIコーディングアシスタント「Cursor」を単なるツールとしてではなく、開発プロセスを加速させる**賢い相棒（Co-pilot）**として最大限に活用することを目的としています。

すべてのエンジニアがCursorの能力を正しく理解し、日々の開発業務をより創造的で生産的なものに変えていくための、基本原則と実践的なノウハウを共有します。

### 🎯 目的
Cursorを使いこなし、コーディング・レビュー・ドキュメント作成のサイクルを高速化する

### 👥 対象読者
開発に携わるすべてのエンジニア

## 2. Cursor利用の基本原則：「AIは副操縦士、機長はあなた」 👨‍✈️
Cursorは非常に強力なツールですが、その性能は私たちエンジニアの使い方に大きく依存します。以下の原則を常に心に留めておきましょう。

### ✅ Cursorのメリット

- **高速なコーディング**: 新規コード、定型コード、テストコードの生成を劇的に高速化します
- **効率的なレビュー**: コードレビューの観点をAIに提案させ、人間のレビューを補強します
- **ドキュメント作成の自動化**: コードコメントや仕様書、READMEの草案を瞬時に作成します

### ⚠️ AIへの心構え

- **思考の主体はあなた**: AIに丸投げするのではなく、あくまで「壁打ち相手」や「アシスタント」として活用し、主体的に思考することをやめないでください。AIへの過度な依存は、長期的に自身のスキルを低下させるリスクがあります
- **最終責任者はあなた**: AIが生成したコードであっても、そのコードをプロダクトに組み込むことを決定するのはあなたです。生成されたコードの品質、動作、セキュリティに関するすべての責任は、最終的に開発者自身にあります

## 3. 【実践編】効果的な使い方とプロンプトエンジニアリング ✍️

Cursorの能力を最大限に引き出すためには、AIとの対話、すなわち「プロンプトエンジニアリング」が鍵となります。

### シーン別ベストプラクティス

| シーン | おすすめのアクション | プロンプトのコツ ✨ |
|--------|---------------------|---------------------|
| コード新規作成 | `Ctrl+L` (チャット) | @で関連ファイルを指定し、要件、機能、期待する動作を明確に伝える |
| リファクタリング | `Ctrl+L` (チャット) | @で対象ファイルを指定し、改善したい点（可読性、パフォーマンスなど）と制約を伝える |
| バグ修正 | `Ctrl+K` (インライン編集) or `Ctrl+L` | バグが起きているコードを選択し、「このコードのバグを修正してください」と依頼。エラーメッセージも添えると精度UP |
| テストコード生成 | `Ctrl+L` (チャット) | @でテスト対象のファイルを指定し、「この関数のテストコードをJestで生成して」のようにフレームワークも指定する |
| ドキュメント/コメント生成 | `Ctrl+K` (インライン編集) | 対象の関数やクラスを選択し、「このコードのJSDocを生成して」「この処理を日本語でコメントアウトして」と依頼 |

### 🚨 UI実装や複雑なロジックを扱う際の注意点

AIはUIの細かなニュアンスや、複雑に絡み合ったビジネスロジックの全体像を一度に理解するのは苦手です。以下の点を意識してください。

#### UI関連タスク

- **責務を分割・明確化**: 「ヘッダー、サイドバー、メインコンテンツを持つレイアウトを作成して」のように、コンポーネントの責務を分割して具体的に指示します
- **デザイン仕様を詳細に**: 「Tailwind CSSを使って、背景はbg-gray-100、ボタンは角丸で影付き(rounded-lg shadow-md)にして」のように、デザインの要件を詳細に記述します。ワイヤーフレームやデザインカンプの情報をテキストで伝えるのがコツです

#### 複雑なロジック

- **一度にすべてを任せない**: 複雑なアルゴリズムやビジネスロジックは、一度にすべてをAIに書かせようとせず、ステップバイステップで対話しながら進めます
- **疑似コードで思考を整理**: まずは日本語や疑似コードで処理の流れを書き出し、それをAIに清書・コード化させるアプローチが有効です

### 高品質なコードを引き出すプロンプト術

優れたプロンプトは、以下の3つの要素で構成されます。

#### 1. コンテキスト（文脈）の提供

- **@ファイル名 or @フォルダ名**: AIに読んでほしいコードを正確に伝えます。これがCursorの最大の強みです
- **@https://...**: 参照してほしい公式ドキュメントや技術ブログのURLを指定できます

#### 2. 制約条件の明示

- **言語・FW**: 「TypeScriptとReactを使ってください」
- **ライブラリ**: 「状態管理はZustandを使用してください」
- **コーディング規約**: 「ESLintのルールに従ってください」「関数名はキャメルケースにしてください」

#### 3. 役割（ペルソナ）の付与

AIに専門家としての役割を与えることで、回答の質と視点が向上します。

**例:**
- 「あなたは経験豊富なシニアセキュリティエンジニアです。以下のコードの脆弱性を指摘してください。」
- 「あなたはUXを最優先するフロントエンドエンジニアです。このコンポーネントをより使いやすく改善する提案をしてください。」

## 4. 【品質担保】安全なコードレビューと変更管理 🛡️

AIの力を借りつつ、コードの品質と安全性を担保するためのルールです。

### AI生成コードのレビューは必須

AIは時々、もっともらしい嘘をつきます（ハルシネーション）。生成されたコードは、必ず人間の目でレビューしてください。

#### 📝 レビュー観点チェックリスト

- [ ] **ロジックの正当性**: 要件を満たし、エッジケースも考慮されているか？
- [ ] **パフォーマンス**: 無駄なループや非効率な処理はないか？
- [ ] **セキュリティ**: XSS、CSRF、SQLインジェクションなどの脆弱性はないか？
- [ ] **可読性・保守性**: 命名規則は適切か？複雑すぎないか？チームのコーディング規約に沿っているか？

### 安全なファイル変更テクニック

#### ブランチは必ず切る
AIにコード変更を依頼する前には、必ず `git checkout -b <branch_name>` で作業ブランチを作成してください。これにより、意図しない変更があっても簡単に元に戻せます。

#### Ctrl+KとCtrl+Lの使い分け

- **`Ctrl+K` (インライン編集)**: 変数名のリネームや、数行の簡単な修正など、小規模な修正に限定します
- **`Ctrl+L` (チャット)**: 機能追加やリファクタリングなど、広範囲に影響する変更はこちらを使い、提示される差分（Diff）を慎重に確認してから適用（Accept）してください

#### （推奨）安全なリファクタリングプロンプト
大規模な変更や既存ロジックの書き換えを依頼する際は、元のファイルを上書きせず、新しいファイルに結果を生成させることで、安全に比較・検討できます。

**プロンプト例:**
```
@feature.ts のリファクタリング案を提案してください。
**元のファイルは変更せず**、結果を **feature_refactored.ts** という新しいファイルに生成してください。
```

## 5. 【チーム活用】ナレッジ共有とアカウント管理 🤝

チーム全体でCursorを使いこなすための仕組みです。

### プロンプトテンプレートの共有

優れたプロンプトはチームの資産です。効果的だったプロンプトは積極的に共有しましょう。

#### 管理方法

- **社内Wiki (Confluenceなど)**: 「Cursorプロンプト集」のような専用ページを作成し、誰でも追記・編集できるようにします
- **Google スプレッドシート**: シーン、プロンプト、解説などをカラムに分けて管理します

#### テンプレート例

##### 新規APIエンドポイント作成用 (Express)

```
あなたはNode.jsとExpressのエキスパートです。
@routes/api.ts
上記のファイルに、以下の仕様で新しいAPIエンドポイントを追加してください。

- エンドポイント: POST /users
- 機能: 新しいユーザーを作成する
- リクエストボディ: { name: string, email: string }
- バリデーション: nameは必須、emailはメール形式であること
- 成功時: 201 Created と user オブジェクトを返す
- 失敗時: 適切なエラーコードとメッセージを返す
```

##### Reactコンポーネント生成用

```
あなたはReactとTypeScriptの専門家です。
以下の仕様で、汎用的なボタンコンポーネント（Button.tsx）を新規に作成してください。

- スタイリング: Tailwind CSS を使用
- Props:
  - variant: 'primary' | 'secondary' | 'danger' (見た目の切り替え)
  - size: 'sm' | 'md' | 'lg'
  - children: React.ReactNode
  - onClick: () => void
- その他: ローディング状態と非活性状態も考慮すること
```

### アカウント管理

- **Proアカウントの更新**: Proアカウントの更新手続きは[担当部署/担当者名]が行います。有効期限が近づいたら、リマインダーが通知されますので、速やかに対応をお願いします
- **期限切れに注意**: Freeプランではコンテキストとして読み込めるファイル数に制限があり、作業が中断する可能性があります。Proアカウントのステータスを定期的に確認してください

## 6. 【スキルアップ】Cursorを自己成長に繋げるために 🌱

Cursorを正しく使えば、自身のスキルアップを加速させることも可能です。

### 学習のポイント

- **鵜呑みにしない、裏付けを取る**: AIの回答はあくまで参考意見です。特に、ライブラリの仕様や関数の使い方については、必ず公式ドキュメント（一次情報）で裏付けを取る習慣をつけましょう
- **「なぜ？」を考える**: AIが生成したコードに対して、「なぜこのコードで動くのか？」「もっと良い書き方はないか？」と自問自答し、自分の言葉で説明できるようになるまで深く理解しましょう
- **新しい技術の学習パートナーに**: 新しい言語やフレームワークを学ぶ際、サンプルコードを書かせたり、エラーの解決策を質問したりする「壁打ち相手」として非常に優秀です

## 7. Cursorをまだ使っていないあなたへ 👋

「まだインストールしていない」「ChatGPTで十分だと思っていた」という方は、ぜひ一度試してみてください。開発体験が大きく変わるはずです。

### 簡単インストール

1. 公式サイトからインストーラーをダウンロード
2. アプリケーションフォルダに移動して起動
3. 初期設定ウィザードに従って設定（キーマップのインポートなどができます）

### ChatGPTとの決定的な違い

Cursorの最大の強みは、あなたのローカル開発環境とシームレスに統合されている点です。

| 比較項目 | ChatGPT | Cursor |
|----------|---------|--------|
| **コードの文脈理解** | コードを都度コピペする必要がある | @シンボルでプロジェクト全体のファイルを瞬時に読み込める |
| **編集・適用** | 生成されたコードを自分でコピペ | 生成された差分をワンクリックで適用・編集可能 |
| **開発フロー** | ブラウザとエディタの行き来が発生 | エディタ内で全ての対話が完結 |

エディタ内でプロジェクト全体を理解したAIと対話しながら開発を進められる体験は、一度味わうと手放せなくなります。まずは簡単なコード生成から試してみましょう！
