# Sample
## 1. 目的・背景
物流企業向けにドライバーの勤怠・配送進捗を一元管理する「DriveeLink」を構築し、手動管理によるコストを 30% 削減する。

## 2. ビジネス要件
| # | 要件 | 根拠・理由 | 優先度 |
|---|------|-----------|--------|
| BR-01 | 月末の勤怠集計を自動で Excel 出力する | 本社総務が 1 日 4h かけて手計算している | 高 |
| BR-02 | 配送進捗をリアルタイム共有 | 顧客問い合わせ 20% 削減 | 中 |

## 3. 機能要件
| # | ユーザーストーリー / 機能 | 受入条件 | 備考 |
|---|--------------------------|----------|------|
| FR-01 | **総務担当者**として、勤怠一覧画面で月次勤怠をダウンロードしたい | ・DL ボタン押下で XLSX が生成<br />・集計誤差 ±0 |  |
| FR-02 | **配車担当者**として、各車両の現在位置をマップで確認したい | ・10 秒以内に地図上にピン表示<br />・位置精度 100m 以内 | GPS API |

## 4. 非機能要件
| 区分 | 指標・目標値 | 備考 |
|------|--------------|------|
| 性能 | 同時接続 500 ユーザーで P95 応答 < 1s | k6 で計測 |
| セキュリティ | OWASP Top10 準拠、CSP/CSRF 対策実装 |  |
| 可用性 | 99.9% / 月 | AWS マルチ AZ |

## 5. ペルソナ
| ペルソナ | 属性 | 目的 | 痛点 |
|----------|------|------|------|
| 佐藤陽子 (35) | 本社総務、勤怠担当 | 月次勤怠を短時間で締めたい | Excel 手計算が煩雑 |
| 鈴木翔 (42) | 配車センター長 | 遅延を即座に把握 | ドライバーから逐次電話が必要 |

## 6. 受入条件
AC-01: XLSX に全従業員分の勤怠が含まれる  
AC-02: 地図ピンが 10 秒以内に更新される

## 7. 付録・用語集
- **配送進捗**：ドライバーがアプリで報告するステータス
