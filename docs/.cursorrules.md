## Setup (Prerequisites)

Before using Cursor Dev Commands, ensure the following setup is completed:

### 1. GitHub CLI Authentication
```bash
# Install GitHub CLI (if not already installed)
# macOS: brew install gh
# Windows: winget install --id GitHub.cli

# Login to GitHub
gh auth login

# Add required scopes for GitHub Projects
gh auth refresh -h github.com -s read:project -s project

# Verify authentication
gh auth status
```

### 2. Required Tools
- **jq**: JSON processor (required for SP script)
  ```bash
  # macOS
  brew install jq
  
  # Linux (Ubuntu/Debian)
  sudo apt-get install jq
  
  # Windows
  winget install jqlang.jq
  ```

### 3. GitHub Projects Configuration
- Create or verify a GitHub Project Board
- Ensure the project has a custom field named **"SP"** or **"Story Points"**
- Field type: TEXT, NUMBER, or SINGLE_SELECT

### 4. Verify Script Files
Ensure the following scripts exist in your project:
- `.cursor/script/setsp.ps` (macOS/Linux)
- `.cursor/script/setsp.ps1` (Windows)

### 5. Required Permissions
- Write access to the repository
- Write access to GitHub Projects

## Custom Commands

This project uses Cursor Dev Commands located in `.cursor/commands/` directory.

Available commands:
- `/issue` - Get GitHub issue information, create branch, and save to issue.md (with optional auto-workflow)
- `/plan` - Create implementation plan
- `/breakdown` - Create individual FE/BE issues based on Plan and calculate Story Points
- `/dev` - Execute development tasks with tests
- `/test` - Run tests and save evidence
- `/pr` - Create pull request

## Development Workflow

### Phase 1: Planning & Issue Breakdown (Assigned to: TL)
Based on the requirement issue created by PM, create branch, plan, calculate SP, and split tasks into FE/BE issues.

1. `/issue {requirement_issue_number}` - Fetch issue, create branch, and document
2. `/plan {requirement_issue_number}` - Create implementation plan
3. `/breakdown {requirement_issue_number}` - Create individual FE/BE issues based on Plan and calculate Story Points

### Phase 2: Development & Deployment Cycle (Assigned to: Individual Engineers)
Triggered by the **FE/BE Issues** created in Phase 1, execute the following development cycle for each issue.

1. `/dev {issue_number}` - Execute development (for FE or BE issue)
2. `/test {issue_number}` - Run tests and collect evidence
3. `/pr {issue_number}` - Create pull request (triggers deployment and manual testing)

---

### Auto-Workflow (New)
*Reflects the workflow sequence (plan → breakdown)*

1. `/issue {issue_number} --auto` - Execute full pipeline automatically (issue → plan → breakdown → dev → test → pr)
2. `/issue {issue_number} --auto --skip-plan` - Skip plan for rapid development
