
## ラベル定義

| ラベル                    | 定義                                        |
| ------------------------ | --------------------------------------------|
| in-dev                   | イシューが開発環境（デベロップメント環境）にある。  |
| in-stg                   | イシューがテスト環境（ステージング環境）にある。    |
| in-prod                  | イシューが本番環境（プロダクション環境）にある。    |
| in-refixing              | イシューがレビュー後の再修正中である。            |
| bug                      | イシューがバグ修正のためのものである。            | 
| feature                  | イシューが新規機能追加のためのものである。       　|
| backend                  | イシューがバックエンドのものである。              |
| frontend                 | イシューがフロントエンドのものである。            |
| high priority            | イシューが至急の対応が必要なものである。           |
| middle priority          | イシューが中程度の優先順位で対応が必要なものである。 |
| low priority             | イシューが不急の対応が必要なものである。           |

## SP定義
- 1 SP = 1 時間 
- 1 タスクに付与できる最大SP = 8 SP = 8 時間 
  - もし、タスクの見積もり時間が8時間を超える場合、そのタスクをより小さなタスクに分解することで、それぞれのタスクが8時間を超えないようにする。

## Label Definition

| Label                    | Definition                                              |
| ------------------------ | ------------------------------------------------------- |
| in-dev                   | The issue is currently in the development environment.  |
| in-stg                   | The issue is currently in the staging environment.      |
| in-prod                  | The issue is currently in the production environment.   |
| in-refixing              | The issue requires a refix after review.                |
| bug                      | The issue is for fixing a bug.                          | 
| feature                  | The issue is for implementing a new feature.            |
| backend                  | The issue is classified as a backend task.              |
| frontend                 | The issue is classified as a frontend task.             |
| high priority            | The issue requires urgent handling.                     |
| middle priority          | The issue requires handling in the ordinary order.      |
| low priority             | The issue can be deprioritized.                         |

## SP Definition
- 1 SP = 1 hour
- Maximum SP per task = 8 SP (8 hours)
  - If a task is estimated to exceed 8 hours, it should be broken down into smaller tasks so that each individual task is less than or equal to 8 SP (8 hours).
